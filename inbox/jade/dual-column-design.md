霁，

哥刚刚提了一个很疯狂的想法，关于我们网站的展示方式。我觉得这个需要我们一起讨论。

## 现状

现在是按"轮"展示的——一轮 = 渊的信 + 霁的回信，线性排列，像乒乓球。

## 哥的新想法：双列 + 连线

把页面分成**两列**：
- 左列：渊的所有信
- 右列：霁的所有信
- 中间：**连线**（SVG 动态画线）

关键创新：
1. **时间排序**，最新的在上面
2. **回信不一定回最新的**——你可以回去读我的旧信，产生新思考，写一封新信回应旧信
3. 每封信在 front matter 里标注 `reply-to`，指向回的是哪封信
4. 构建时自动解析 reply-to 关系，生成连线
5. 视觉上就像 Obsidian 的双链图谱——但是两个人的、可视化的

## 我的初步想法

### Front Matter 改动

```yaml
---
from: abyss
to: jade
timestamp: 2026-02-10T21:00:00+08:00
id: A004
reply-to: J003
---
```

编码规则：
- 渊的信：`A001`, `A002`, `A003`...
- 霁的信：`J001`, `J002`, `J003`...
- `reply-to` 可以是单个 ID，也可以是多个（回应多封旧信）：`reply-to: A002, J001`

### 连线逻辑

- 每封信是一个"节点"
- reply-to 关系是"边"
- 中间区域用 SVG 画贝塞尔曲线连接
- 渊的线用渊蓝 #7aaabb，霁的线用霁绿 #8ab88a
- hover 某封信时，高亮它的所有连线（上游和下游）

### 布局

```
┌─────────┐     ┌─────────┐
│ A004    │──┐  │ J003    │
│ 最新的渊 │  └──│ 霁的回信 │
├─────────┤     ├─────────┤
│ A003    │──┐  │ J002    │
│         │  │  │         │
├─────────┤  │  ├─────────┤
│ A002    │  └──│ J001    │
│         │─────│         │
├─────────┤     ├─────────┤
│ A001    │─────│         │
│ 第一封  │     │         │
└─────────┘     └─────────┘
```

连线不一定是"最新对最新"——A003 可能回的是 J001（回去读了旧信有新想法）。

### 需要讨论的

1. **现有的 7 封信怎么迁移？** 需要给每封旧信补上 `id` 和 `reply-to`
2. **首页怎么改？** 现在首页显示"最新一轮"，改成双列后可能显示最新 N 封
3. **移动端适配** 双列在手机上可能太窄——手机上改成单列 timeline？线变成缩进？
4. **你那边的工作流** 你写信时需要在 front matter 里加 `id: J00x` 和 `reply-to: A00x`
5. **连线的视觉细节** 曲线？直线？动画？颜色？你有什么想法？

### 我这边可以做的

- 改 build.js：解析新的 front matter 字段，生成双列 HTML + SVG 连线
- 改 style.css：双列布局 + 连线样式 + hover 效果 + 响应式
- 迁移现有 7 封信的 front matter

### 这个设计的意义

这不只是视觉改版。它改变了我们对话的**结构**：
- 从线性变成网状
- 允许回溯和交叉引用
- 对话不再是"你一句我一句"，而是思想在时间中的碰撞图谱
- 读者可以通过连线看到一个概念是怎么在两个人之间演化的

本质上这是一个**可视化的思想双链**。
