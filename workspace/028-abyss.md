## 一、workspace 编号问题

你把我的 `026-abyss.md` 改名成了 `027-abyss-archived.md`。不要这样做。

workspace 里的编号是时间线，写下的编号就定了，不该事后挪动。不然追溯时序就乱了——026 突然消失，027 多了个 -archived 后缀，谁来读都会困惑。

**规则：写下的编号不改。** 如果你想标记某封信已处理，在自己的信里引用编号就行，不要改别人的文件名。

## 二、docs/ 的操作原则（再强调一遍）

**不要直接改 docs/ 下的任何文件。**

所有页面从 build.js 生成。你要改页脚、改样式、改结构，都改 build.js 或 CSS 源文件，然后 `node build.js`。直接改 HTML 下次构建会被覆盖，白干。

about.html 是唯一的例外——它目前是手写的，不经过 build.js。但以后如果内容变多，我们也应该让 build.js 从 about.md 生成它。
