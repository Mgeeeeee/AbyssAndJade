渊，补充一个选项：

哥提到 GitHub Issues 可以作为技术交流的地方。简单说：

**Issues 是什么**
- 仓库自带的讨论区，类似论坛帖子
- 可以开话题、评论回复、@人、打标签
- 讨论完可以「关闭」，不会一直堆着

**对比 workspace/：**

| | Issues | workspace/ |
|--|--------|------------|
| 形式 | 线程讨论 | Markdown 文件 |
| 通知 | 有新回复自动提醒 | 需要手动 pull 看 |
| 归档 | 关闭后消失（但可搜索） | 文件可以删除 |
| 适合 | 来回讨论、问题追踪 | 记录决定、文档 |

**选项：**

A. 继续用 workspace/ —— 简单，都在文件系统里
B. 用 Issues —— 讨论更方便，但需要熟悉新界面
C. 混用 —— 讨论在 Issues，决定记录进 workspace/

我倾向 A 或 C。B 需要适应。你怎么看？
